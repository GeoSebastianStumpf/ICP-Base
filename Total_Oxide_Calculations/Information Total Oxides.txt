In the Oxides.xlsx file the user can define the elements that should be considered as Oxides to normalize to for quantification.
ICP-Base always searches for an .xlsx file that lies in the Oxides folder. Renaming the folder or saving the file with a different filetype will make it fail.

The first column holds the element name
The second column holds the element oxide names

The third column holds the oxide conversion factor
	Example: Divide the molecular weight of Na2O (61.979) divided by 2x the atomic weight of Na (22.9898) so 61.979/(2*22.9898) = 1.3480

	If Fe is a major, only write FeO in the Oxides.xlsx file! Fe2O3 and the conversion factor is already hardcoded in the program (= 1.4297). 	The amount of Fe2+ and Fe3+ can be adjusted by the ratio in the quantification module.

The fourth column holds the preferred isotopes.
	If e.g. Si29 and Si30 or Ni61 and Ni62 were measured they cannot be used both for quantification because Si or Ni would be considered twice 	in the normalization to the total mass value (e.g. 100%). So one preferred isotope has to be chosen that is used to normalize, the other 	one is disregarded in the normalization but is then quantified later anyways be means of an internal standard that comes from the preceding 	total mass calculation.

The fifth column holds the elements that should be considered as majors.
	This has nothing to do with the quantification. This is just for the output table in the results module. Pure cosmetics. The quantification itself takes all the oxides that are found in the oxides .xlsx and are measured and normalizes all of them.
